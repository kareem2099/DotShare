# 🚀 DotShare v3.1.0: Technical Reconstruction Guide

This document serves as a comprehensive technical log and developer guide for the **v3.1.0 "Polish Pass"** update. It details the architectural improvements, UI/UX systems, and critical bug fixes implemented to transform DotShare into a production-grade VS Code extension.

---

## 🏗️ 1. Architecture Overview: The WebView Bridge

The DotShare UI operates inside a **VS Code WebView**, communicating with the extension backend via a `postMessage` bridge. 

*   **Frontend (WebView)**: `media/webview/app.ts` handles UI interactions, state management, and toast notifications.
*   **Backend (Extension)**: `src/handlers/MessageHandler.ts` acts as the primary router, delegating tasks to specialized handlers like `PostHandler` or `ConfigHandler`.

### Message Life Cycle
1.  User clicks **"Share"** → WebView sends `share` command with payload.
2.  `MessageHandler` receives the command → routes to `PostHandler`.
3.  `PostHandler` executes async sharing logic → sends `status` updates (info/success/error).
4.  Upon completion, backend sends `shareComplete` → WebView triggers a **UI Reset**.

## 🎨 2. UI/UX Systems (The "Polish" Pass)

### A. The Toast Notification System
Replaced standard VS Code popups with a custom, non-intrusive notification system inside the WebView.
*   **File**: `media/webview/app.ts` & `media/webview/style.css`
*   **Features**: Progress bars, auto-dismiss, and contextual colors.

```typescript
// app.ts: The toast function
function toast(msg: string, type: 'success' | 'error' | 'warning' | 'info' = 'info', ms = 5000): void {
    const container = get('toast-container');
    if (!container) return;
    const el = document.createElement('div');
    el.className = 'toast ' + type;
    el.innerHTML = `
        <span>${icons[type]}</span>
        <span>${escHtml(msg)}</span>
        <div class="toast-progress"></div>
    `;
    container.appendChild(el);
    if (ms > 0) setTimeout(() => el.remove(), ms);
}
```

### B. Global Loading States
To prevent duplicate submissions and provide visual feedback, every primary action now has a "Loading" state.
*   **Logic**: Buttons switch to a "Spinning/⏳" state and disable themselves during async operations.

## 📎 3. The Media Engine & Critical Fix

### The "Disappearing Preview" Race Condition
**Bug**: When a user attached a file, the preview would appear for a split second and then vanish.
**Root Cause**: The backend sent a "File uploaded successfully" success message. The WebView's message handler was programmed to `resetAllComposers()` whenever it received *any* success status containing the word "success". 
**The Fix**: Implemented **Selective Reset Logic**. The WebView now distinguishes between *terminal* success messages (Sharing complete) and *intermediate* success messages (File uploaded).

```typescript
// app.ts: Refined Status Handler
case 'status':
    if (msg.type === 'success') {
        const statusLower = String(msg.status || '').toLowerCase();
        const isCompletion = statusLower.includes('success') || statusLower.includes('shared');
        const isIntermediate = statusLower.includes('upload') || statusLower.includes('save');

        // Only reset UI if it's a completion message, NOT an upload success
        if (isCompletion && !isIntermediate) {
            resetAllComposers();
        }
    }
    break;
```

## 🐛 6. Critical Bug Fixes: v3.1.0 Final Polish

### A. Character Limit Validation System

**Issue**: The `MAX_CHARS` constant was defined but never used, wasting a platform-aware character limit feature.

**Solution**: Integrated character limit validation into `updateCharCounter()` and `updateShareBtn()` functions.

**Location**: `media/webview/app.ts` (lines 395-422)

```typescript
const MAX_CHARS: Record<string, number> = {
    x: 280, bluesky: 300, linkedin: 3000, telegram: 4096, 
    facebook: 63206, discord: 2000, reddit: 40000,
    devto: 100000, medium: 100000,
};

function updateCharCounter(): void {
    if (!textarea || !counter) return;
    const len = textarea.value.length;
    const platform = activeCommandPlatform || '';
    const maxChars = platform && MAX_CHARS[platform] ? MAX_CHARS[platform] : null;
    
    // Display "current/max" format for platforms with limits
    counter.textContent = maxChars ? `${len}/${maxChars}` : String(len);
    counter.className = 'compose-counter';
    
    // Apply warning/error classes based on character limit
    if (maxChars) {
        const warnThreshold = Math.floor(maxChars * 0.8);
        const errorThreshold = maxChars;
        if (len > errorThreshold) {
            counter.classList.add('error');
        } else if (len > warnThreshold) {
            counter.classList.add('warn');
        }
    }
}

function updateShareBtn(): void {
    if (!btnShare) return;
    if (!textarea?.value.trim().length) {
        btnShare.disabled = true;
        return;
    }
    // Disable if exceeds max chars for the platform
    const platform = activeCommandPlatform || '';
    const maxChars = platform && MAX_CHARS[platform] ? MAX_CHARS[platform] : null;
    btnShare.disabled = maxChars ? textarea.value.length > maxChars : false;
}
```

**Benefits**:
- Visual feedback when approaching platform limits (warning at 80%)
- Error styling when exceeding limits
- Share button automatically disables for over-length posts
- Per-platform character counting (X: 280, LinkedIn: 3000, Reddit: 40000, etc.)

### C. Reddit Post Sharing: Field Name Synchronization

**Issue**: The `shareRedditPostBtn` event listener was sending incorrectly named fields that didn't match the `PostHandler` expectations:

**Mismatches**:
- ❌ `subreddit` → ✅ `redditSubreddit`
- ❌ `title` → ✅ `redditTitle`
- ❌ `flair` → ✅ `redditFlairId`
- ❌ `postType` → ✅ `redditPostType`
- ❌ `spoiler` → ✅ `redditSpoiler`
- ❌ **Missing** `post` (the actual post text)

**Location**: `media/webview/app.ts` (lines 1270-1285)

```typescript
get('shareRedditPostBtn')?.addEventListener('click', () => {
    try {
        const val = (id: string) => (get<HTMLInputElement>(id)?.value ?? '').trim();
        const postText = textarea?.value.trim() || '';

        send('shareToReddit', {
            post:            postText,                    // ✅ Now included
            redditSubreddit: val('redditSubreddit'),      // ✅ Correct field name
            redditTitle:     val('redditTitle'),          // ✅ Correct field name
            redditFlairId:   get<HTMLSelectElement>('redditFlair')?.value || '',  // ✅ Uses .value directly
            redditPostType:  document.querySelector<HTMLInputElement>('input[name="redditPostType"]:checked')?.value || 'self',  // ✅ Correct field name
            redditSpoiler:   get<HTMLInputElement>('redditSpoiler')?.checked || false  // ✅ Correct field name
        });
        const modal = get('redditPostModal');
        if (modal) modal.style.display = 'none';
        toast('Sharing to Reddit…', 'info');
    } catch (error) {
        console.error('Share to Reddit error:', error);
        toast('Failed to share to Reddit', 'error');
    }
});
```

**Expected Handler Signature** (`src/handlers/PostHandler.ts`):
```typescript
const {
    post,
    redditSubreddit,
    redditTitle,
    redditFlairId,
    redditPostType,
    redditSpoiler
} = msg;
```

**Impact**:
- Reddit posts now include the actual post text (previously missing)
- Field names align with backend handler expectations
- Flair selection uses `.value` instead of raw element
- Radio button state properly extracted for post type
